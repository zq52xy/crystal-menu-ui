# Prompt: Generate JRPG Menu UI

Build a learning-only classic JRPG menu interface with original assets.

Use these components:

- `MenuPanel`
- `MainMenu`
- `CommandButton`
- `DialogueBox`
- `FloatingStatusBar`
- `EquipmentPanel`
- `OrbGem`
- `OrbSocketRail`
- `PartyMenuShell`
- `ConfirmDialog`
- `SaveSlot`
- `GameIcon`
- `CharacterProfile`
- `CharacterPortrait`
- `CharacterRoster`
- `ProfileScreen`
- `Divider`
- `Loading`
- `InventoryList`
- `ItemTooltip`
- `BattleMenu`
- `PartyStatus`
- `HPMPBar`
- `LevelInfo`

No planned components remain from the current source-method roadmap.

Visual direction:

- dark glass menu panels,
- thin cyan borders,
- white primary text,
- cyan active highlight,
- angular level and status guide lines,
- pulsing selected main-menu rows,
- bottom playtime/currency/hint/action metadata,
- equipment slots with compact stat deltas and linked orb sockets,
- standalone CSS-rendered orb gems and large blue orb socket rails with colored gems and brace connectors,
- reusable party/menu screen shell with command, primary, detail, and footer regions,
- centered glass confirmation popup with metadata, message, and two actions,
- save/load archive rows with timestamp, location, playtime, party initials, and selected/autosave/empty states,
- original cyan HUD icons for combat, magic, defense, stat, equipment, time, currency, and summon categories,
- reference-image-style character profile panels with optional authorized portrait images,
- reusable portrait frames, selectable character rosters, and full profile screen composition,
- compact HUD dividers and crystal-ring loading states,
- dense inventory rows, compact item tooltips, and battle command shells,
- dense status information,
- compact command lists,
- no official game assets,
- no direct clone of any known screen.

The UI should feel like a modern JRPG Remake HUD system while remaining original and reusable.
