# Module Map: eval

## Purpose

Quality contracts, evidence reports, and visual QA artifacts for the study kit.

## Members

- `visual-contract.md`: durable visual quality rules.
- `work-contract.md`: current sprint done definition.
- `evidence-report.md`: implementation evidence.
- `publish-evidence-report.md`: GitHub publishing preparation and blocker evidence.
- `visual-smoke.json`: generated browser smoke-test report.
- `screenshots/`: generated desktop, mobile, component, and README hero review screenshots.

[PROTOCOL]: 变更时更新此头部，然后检查 CLAUDE.md

## Boundaries

- Evidence records what happened; it must not be edited to hide failures.
- Screenshots are evidence until a human explicitly accepts them as baselines.
